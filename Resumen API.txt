Una API (interfaz de programación de aplicaciones) Las API son un medio simplificado para conectar su propia infraestructura a través del desarrollo de aplicaciones nativas de la nube, pero también te permiten compartir tus datos con clientes y otros usuarios externos. Las API públicas aportan un valor comercial único porque simplifican y amplían tus conexiones con los partners y, además, pueden rentabilizar tus datos.

Si la API es privada, solo se puede usar internamente, dando  las empresas un mayor control sobre ellas. De partners, se comparten con otras empresas, da flujos de ingresos adicionales, sin comprometer la calidad.Las públicas,todos tienen acceso a la API, sirven como un recurso para innovar e interactuar. Las APi remotas interactúan por remoto, los recursos que administran las API, están fuera de la computadora que solicita alguno de los recurso. La red de comunicaciones más usada es Internet, la mayoría de las API están diseñadas de acuerdo con los estándares web. No todas las API remotas son API web, pero las API web son remotas.Las API web normalmente usan HTTP para solicitar mensajes y proporcionar una definición de la estructura de los mensajes de respuesta. Estos mensajes de respuesta toman la forma de un archivo XML o JSON.Protocolo de Acceso a Objetos Simples (SOAP). Las API diseñadas con SOAP usan XML para el formato de sus mensajes y reciben solicitudes a través de HTTP o SMTP. Con SOAP, las aplicaciones que funcionan en entornos distintos o están escritas en diferentes lenguajes comparten información.

Otra especificación es la Transferencia de Estado Representacional (REST). Las API web que funcionan con las limitaciones de arquitectura REST se llaman API de RESTful. La diferencia más básica entre ambas es que SOAP es un protocolo, mientras que REST es un estilo de arquitectura. 

Los dos enfoques de arquitectura que más se utilizan para las API remotas son la arquitectura orientada al servicio (SOA) y la arquitectura de microservicios.Aunque en muchos aspectos la SOA es más sencilla que una arquitectura monolítica, conlleva un riesgo de cambios en cascada en todo el entorno si las interacciones de los componentes no se comprenden claramente.

Un webhook es una función de devolución de llamadas que se basa en el protocolo HTTP para que dos API se comuniquen mediante eventos de forma ligera.
